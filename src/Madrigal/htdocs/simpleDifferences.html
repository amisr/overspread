How is the simple data access interface different from the standard Madrigal interface?

The simple Madrigal web interface is designed for a user who is using Madrigal for the first time, or only requires access to the most basic parameters.

When printing data, the user only sees data in the Madrigal file that has associated error measurements. No parameters without associated error measurements are shown, and no derived parameters can be displayed. No filtering of the data is possible - the Print data page displays every record in the time ranges selected.

When plotting data, the user can only choose to plot parameters that have associated error measurements. No parameters without associated error measurements and no derived parameters can be selected. No filtering of the data is possible - the Plot data page displays every record in the time ranges selected

To move beyond these limitiations, users should select some other data access method from the data access page. A tutorial on these ways of accessing Madrigal data is available here.
